Сценарии

ZAP поддерживает сценарии, которые могут быть встроены в ZAP и могут обращаться к внутренним структурам данных и классам ZAP. Эти сценарии позволяют динамически улучшать ZAP из ZAP.

ZAP supports any scripting language that supports JSR 223, including:

  • ECMAScript / JavaScript (through the GraalVM JavaScript add-on)
  • Zest
  • Groovy https://groovy-lang.org/
  • Kotlin https://kotlinlang.org/
  • Python https://www.jython.org
  • Ruby - https://jruby.org/
  • и многое другое ...
ВНИМАНИЕ - сценарии запускаются с теми же разрешениями, что и ZAP, поэтому не запускайте сценарии, которым вы не доверяете!

Тип сценария

Поддерживаются разные типы скриптов:
  • Активные правила - они запускаются как часть активного сканера и могут быть включены индивидуально.
  • Аутентификация - скрипты, которые вызываются, когда аутентификация выполняется для контекста. Чтобы их можно было использовать, их необходимо выбрать при настройке метода аутентификации на основе сценария для контекста.
  • Extender - скрипты, которые могут добавлять новые функции, включая графические элементы и новые конечные точки API.
  • HTTP Sender - скрипты, которые запускаются для каждого запроса / ответа, отправленного / полученного ZAP. This includes the proxied messages, messages sent during active scan, fuzzer, ...
  • Входной вектор - скрипты для определения того, что именно ZAP должен атаковать. Они также могут изменить способ представления любого URL-адреса в дереве сайтов.
  • Пассивные правила - они работают как часть пассивного сканера и могут быть включены индивидуально.
  • Прокси-сервер - они запускаются «встроенными», могут изменять каждый запрос и ответ и могут быть включены индивидуально. Они также могут вызывать точки останова
  • Управление сеансом - скрипты, определяющие, как приложение управляет сеансами.
  • Stand Alone - автономные скрипты, которые запускаются только тогда, когда вы запускаете их вручную.
  • Целевые - скрипты, которые вызываются с целевым URL-адресом и запускаются только тогда, когда вы запускаете их вручную.
Примечание: Дополнения могут добавлять дополнительные типы скриптов, которые должны быть описаны в справке соответствующего дополнения.

Для получения дополнительных сведений о том, как запускать сценарии ZAP, см. Страницы справки надстройки Script Console.
