トップレベルツールバー

このツールバーは、一般的に使用される機能を提供します。

モードプルダウン

現在のモードを変更できます。

 新規セッション

このボタンは ファイルメニュー の「新規セッション」メニュー項目と同じです。

  セッションを開く

このボタンは ファイルメニューの「セッションを開く」メニュー項目と同じです。

  セッションを保存…

このボタンは、ファイルメニューの「セッションを保存…」メニュー項目と同じです。

  スナップショット セッション

このボタンは、 ファイルメニューの「スナップショット セッション」メニュー項目と同じです。

  セッションプロパティ...

このボタンは、ファイルメニュー の「プロパティ...」メニュー項目と同じです。

  オプション...

このボタンは、ツールメニュー の「オプション...」メニュー項目と同じです。

  ルック＆フィールを動的切り替え

ZAPデスクトップUIの外観を定義するルック・アンド・フィールを動的に切り替えることができ、「ダークモード」のサポートも含まれています。

  全てのタブを表示

このボタンは隠されているタブをすべて表示します。

  ピン留めされていないタブを非表示

このボタンは「ピン留めされていない」すべてのタブを非表示にします。 タブは、選択時に表示される小さな「ピン」アイコンで固定および固定解除できます。

  タブ名とアイコンを表示

このボタンはタブ名の表示を切り替えます。

  サイトタブを展開

これにより表示が変更され、サイトタブを含む「ツリー」ウィンドウが左側の全長に拡張されます。
これにより「情報」ウィンドウで利用可能な領域が縮小されます。

  情報タブを展開

これにより表示が変更され、「情報」ウィンドウが下部の全長に拡張されます。
これにより「ツリー」ウィンドウで利用可能な領域が縮小されます。

  フルレイアウト

これにより表示が変更され、選択されたタブが全画面を占有します。
これは小さな画面でZAPを使用する際に便利です。

  リクエストとレスポンスのタブを左右に並べて表示

これにより表示が変更され、リクエストタブとレスポンスタブが横並びになります。
これにより表示できる情報量は増加しますが、リクエストとレスポンスを同時に見ることはできません。

  リクエストとレスポンスのパネルを同じタブ内で左右に並べて表示

これにより表示が変更され、リクエストパネルとレスポンスパネルが同じタブ内で横並びに表示されます。
これにより表示できる情報量は減少しますが、リクエストとレスポンスを同時に見ることができます。

  リクエストをレスポンスの前に表示

これにより表示が変更され、リクエストパネルがレスポンスパネルの前に表示されます。
これにより表示できる情報量は減少しますが、リクエストとレスポンスを同時に見ることができます。

  リクエストとレスポンスのパネルを左右に並べて表示

これにより表示が変更され、リクエストパネルがレスポンスパネルの左に表示されます。
これにより表示できる情報量は減少しますが、リクエストとレスポンスを同時に見ることができます。

 /   すべてのリクエストとレスポンスでのブレーク設定 / 解除

これにより、次のリクエストまたはレスポンスをトラップし、ブレークタブに表示する「グローバル」ブレークポイントを設定および解除します。
その後、リクエストまたはレスポンスの任意の部分を変更し、「ステップ」または「続行」ボタンを押すことで対象アプリケーションに送信できます。
または、「破棄」ボタンを押してリクエストまたはレスポンスを破棄することもできます。
オプション ブレークポイント画面で、単一の「統合」ブレークボタンと、リクエスト用とレスポンス用の個別ボタンを切り替えることができます。

 /   すべてのリクエストでのブレーク設定 / 解除

これにより、次のリクエストをトラップし、ブレークタブに表示する「グローバル」ブレークポイントを設定および解除します。
その後、リクエストの任意の部分を変更し、「ステップ」または「続行」ボタンを押すことで対象アプリケーションに送信できます。
または、「破棄」ボタンを押してリクエストを破棄することもできます。
オプション ブレークポイント画面で、単一の「統合」ブレークボタンと、リクエスト用とレスポンス用の個別ボタンを切り替えることができます。

 /   すべてのレスポンスでのブレーク設定 / 解除

これにより、次のレスポンスをトラップし、ブレークタブに表示する「グローバル」ブレークポイントを設定および解除します。
その後、レスポンスの任意の部分を変更し、「ステップ」または「続行」ボタンを押すことでブラウザに送信できます。
または、「破棄」ボタンを押してレスポンスを破棄することもできます。
オプション ブレークポイント画面で、単一の「統合」ブレークボタンと、リクエスト用とレスポンス用の個別ボタンを切り替えることができます。

  ステップ

これにより、トラップされたリクエストまたはレスポンスを、加えた変更と共にアプリケーションまたはブラウザに続行させることができます。
「グローバル」ブレークポイントは設定されたままなので、次のリクエストまたはレスポンスもキャッチされます。
このボタンはリクエストまたはレスポンスがトラップされた時のみ有効になります。

  続行

「グローバル」ブレークポイントが解除され、特定のURLにブレークポイントを設定していない限り、後続のリクエストとレスポンスはZAPによってキャッチされなくなります。
このボタンはリクエストまたはレスポンスがトラップされた時のみ有効になります。

  破棄

これによりトラップされたリクエストまたはレスポンスが破棄され、アプリケーションやブラウザに渡されません。
このボタンはリクエストまたはレスポンスがトラップされた時のみ有効になります。

  カスタムHTTPブレークポイントを追加

これによりブレークポイントの追加ダイアログが表示され、ブレークポイントの条件を指定できます。

  スキャンポリシー管理…

スキャンポリシー管理ダイアログを表示し、スキャンポリシーの設定ができます。

  アドオンの管理

これによりアドオンの管理ダイアログが表示され、オンラインマーケットプレイスからアドオンを発見、インストール、更新できます。
アドオンのアンインストールも可能です。

 /   強制ユーザーモード オン / オフ

強制ユーザーモードのオン・オフを切り替えます。
このボタンは、少なくとも1つのコンテキストに対して強制ユーザーを定義した場合のみ有効になります。これはセッションコンテキストダイアログから実行できます。

アドオンにより追加のボタンが追加される場合があります。
