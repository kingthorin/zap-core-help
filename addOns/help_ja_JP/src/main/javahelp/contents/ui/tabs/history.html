履歴タブ

履歴タブは、すべてのリクエストを作成順に一覧表示します。
各リクエストについて以下の情報を確認できます：
     リクエストインデックス - 各リクエストには1から始まる番号が付けられます
     HTMLメソッド、例：GETまたはPOST
     リクエストされたURL
     HTTPレスポンスコード
     HTTPレスポンスコードの意味の簡単な要約
     リクエスト全体にかかった時間の長さ
     リクエストに関するアラート
     リクエストに追加したノート
     リクエストに関するタグ
リクエストを選択すると、上部の リクエストタブ と レスポンスタブ に表示されます。

フィルターツールバー

表示するリクエストを制限できるフィルタツールバーが提供されています。
フィルタボタンをクリックすると 履歴フィルタダイアログが表示されます.
現在適用されているフィルタの概要がボタンの右側に表示されます。

右クリック メニュー

ノードを右クリックすると、以下の操作を可能にするメニューが表示されます：

攻撃

攻撃メニューには以下のサブメニューがあります：

アクティブスキャン…

これにより アクティブスキャン ダイアログが起動し、選択したリクエストを開始点として アクティブスキャン を開始できます。

スパイダー...

これによりスパイダーダイアログが起動し、選択したリクエストを開始点としてスパイダーを開始できます。

コンテキストに含める

このメニューでは、選択したノードとそのすべての下位要素を指定した コンテキストに含めることができます.
新しいコンテキストを作成するオプションもあります。
セッションコンテキスト ダイアログが表示され、追加の変更を行うことができます。

コンテキストから除外

このメニューでは、選択したノードとそのすべての下位要素を指定した コンテキストから除外できます.
セッションコンテキスト ダイアログが表示され、追加の変更を行うことができます。

コンテキストとして指定

このメニューには、定義した各 コンテキスト に対して以下のサブメニューがあります：

コンテキスト名 : フォームベース認証ログインリクエスト

指定したノードを指定コンテキストのログインリクエストとして識別します。
1つのコンテキストでそのように識別できるノードは1つのみです。
セッションコンテキスト認証 画面が表示され、追加の変更を行うことができます。

コンテキスト名 : JSONベース認証ログインリクエスト

指定したノードを指定コンテキストのログインリクエストとして識別します。
1つのコンテキストでそのように識別できるノードは1つのみです。
セッションコンテキスト認証 画面が表示され、追加の変更を行うことができます。

コンテキスト名 : データ駆動型ノード

指定したノードを指定コンテキストの データ駆動型コンテンツ として識別します。
セッションコンテキスト構造 画面が表示され、追加の変更を行うことができます。

以下の処理から除外する:

このメニューには以下のサブメニューがあります：

プロキシ

選択したノードをプロキシから除外します。ZAP経由でプロキシされますが、どのタブにも表示されません。
現在テストしているシステムに関係のないURLを無視するために使用できます。
ノードは セッションプロパティ ダイアログから再度含めることができます。

スキャナー

選択したノードがアクティブスキャンされることを防ぎます。
ノードは セッションプロパティ ダイアログから再度含めることができます。

スパイダー

選択したノードがスパイダーされることを防ぎます。
ノードは セッションプロパティ ダイアログから再度含めることができます。

履歴タグの管理…

これにより 履歴タグの管理ダイアログ が表示され、リクエストに関連付けられた タグ を変更できます。

履歴IDに移動…

これによりダイアログが表示され、ジャンプしたい履歴エントリのID (番号) の入力を求められます。 入力されたID値が表示されている履歴項目でない場合、そのエントリより上の最も近いIDが表示されます。リストの終端を超えている場合は、テーブルの最後の項目が表示されます。 Ctrl + Alt + J でもアクセスできます。

ノート…

これにより ノートの追加ダイアログ が表示され、リクエストに関連する ノート を記録できます。

削除

これによりノードとそのすべての子要素がZAPから削除されます。
ただし、それらを再度追加することは可能です。これを防ぐには「除外」メニューを使用してください。

ブレーク…

これにより ブレークポイントの追加ダイアログ が表示され、そのURLにブレークポイントを設定できます。

このノードのアラート

選択したURLに関連付けられた アラート がある場合、このメニューの下に一覧表示されます。
アラートの1つを選択すると、そのアラートが表示されます。

新しいアラート...

これにより アラートの追加ダイアログ が表示され、このリクエストに対して新しい アラート を手動で記録できます。

サイトタブで表示

選択したメッセージを サイトタブに表示します.

URLをブラウザで開く

選択したノードのURLを既定のブラウザーで開きます。

Anti-CSRFテストフォームを生成

これによりCSRF問題のテスト用に生成されたフォームを提供するURLが開きます。
この機能は、APIが有効化されていて、かつお使いのプラットフォームでJavaがブラウザでURLを開くことをサポートしている場合に限り、POSTリクエストに対して有効になります。
