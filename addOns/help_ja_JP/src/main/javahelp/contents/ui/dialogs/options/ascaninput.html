オプション アクティブスキャン入力ベクター画面

この画面では アクティブスキャン の入力ベクターを設定できます。
これらは、アクティブスキャナーが攻撃する要素です。
サポートされているすべての要素をスキャンすると時間がかかりますが、一部の要素をスキャンしないと、いくつかの脆弱性が見逃される可能性があります。

インジェクション可能なターゲット

アクティブスキャナーがターゲットとするリクエスト要素：

URLクエリ文字列とデータ駆動型ノード

リクエストURLのクエリにおける、キーと値のペア、すなわち次の部分: ?.
もし データ駆動型ノード がコンテキスト内で定義されている場合、それらはテストされます。
URLクエリパラメーターを追加しますか?
ZAPは元々クエリパラメータを持たないGETリクエストに、クエリパラメータを追加します。これにより、スキャンに時間がかかる可能性がありますが、見過ごされる可能性のある脆弱性を発見できるかもしれません。

POSTデータ

リクエストPOSTデータにおける、キーと値のペア。

プレーンボディデータ

ZAPはこのオプションが選択されると、ボディコンテンツを持ち、Content-Type が設定されていないか、text/plain であるリクエストを、テスト目的で単一の入力/パラメータとして処理します。

URLパス

リクエストURLのパスにおける、次の文字で区切られた要素: /.

HTTPヘッダー

リクエストHTTPヘッダー
すべてのリクエスト
クエリやリクエストボディを介してパラメータを送信するリクエストだけでなく、すべてのリクエストのHTTPヘッダーをスキャンすることを許可します。

Cookieデータ

リクエスト Cookie
Cookie値をエンコード
このオプションを選択すると、Cookieの値は攻撃時にURLエンコードされます。デフォルトではエンコードされません。これはZAP 2.15.0からの新しい挙動で、以前は常にエンコードされていました。

組み込み入力ベクターハンドラー

アクティブスキャナーがターゲットとするデータ形式：
    マルチパートフォームデータ
    XMLタグ/属性
    JSON
    Google Web Toolkit
    OData ID/フィルター

スクリプト入力ベクターを有効化

このオプションを選択すると、アクティブスキャナーは有効になっているスクリプト入力ベクターを使用します。
スクリプト入力ベクターとは、ZAPにあなたが記述またはインポートしたスクリプトであり、デフォルトでサポートされていない要素をターゲットにすることができます。

この画面では、アクティブスキャナーが無視するパラメーターを設定することもできます。
パラメーター名に関する詳細は、「 パラメーター セクションの「 アラートの追加 」ダイアログを参照してください。
