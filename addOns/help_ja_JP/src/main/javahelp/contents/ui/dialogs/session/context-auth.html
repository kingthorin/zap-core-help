セッションコンテキスト認証画面

これはセッションコンテキスト画面の一つで、コンテキストの認証方式を管理できます。

認証方式

認証方式の種類を選択後、設定が必要なオプションは認証方式によって異なります。

注意: ユーザーが定義された後に認証方式を変更すると、認証方式に一致するユーザー認証情報の種類が必要なため、認証情報がリセットされる場合があります。 この場合、確認ダイアログが表示されます。

手動認証

この認証方式では設定は不要です。 詳細...

フォームベース認証

この認証方式を設定するには、ログインリクエストが実行される ログインURL、必要に応じて リクエストボディ (POSTデータ)、および「ユーザー名」と「パスワード」を指定するために使用される パラメーター を入力する必要があります。 リクエストボディが提供されない場合、ログインリクエストはHTTP GETとして実行され、そうでない場合はHTTP POSTが使用されます。 認証情報自体は ユーザー タブで設定します。 ログインページ を指定して、新しいセッション (例: Cookie) を取得し、リクエストボディに含まれる Anti-CSRFトークン を再生成する元となる場所を示すこともできます。 ログインページ が提供されない場合は、 ログインURLが使用されます.

JSONベース認証

この認証方式を設定するには、ログインリクエストが実行される ログインURL、 JSONオブジェクト (POSTデータ、 application/json)、および「ユーザー名」と「パスワード」を指定するために使用される パラメーター を入力する必要があります。認証情報自体は ユーザー タブで設定します。 ログインページ を指定して、新しいセッション (例: Cookie) を取得する元となる場所を示すこともできます。 ログインページ が提供されない場合は、 ログインURLが使用されます.
詳細...
POSTデータの例：
  • {"username":"{%username%}","password":"{%password%}"}
  • {"user":{"mail":"{%username%}","password":"{%password%}"}}
ここで、 {%username%} と {%password%} は認証情報が設定される場所を示します。

HTTP/NTLM認証

この認証方式を設定するには、認証が行われるサーバーの ホスト名 と ポート 、および認証情報が適用される レルム を指定する必要があります。認証情報自体は ユーザー タブで設定します。 詳細...

スクリプトベース認証

この認証方式を使用するには、 まず スクリプト タブを使用して 認証スクリプト を作成 (および保存) する必要があります (スクリプトタブにはこのタイプのテンプレートや例が用意されています)。 次に、ドロップダウンリストでスクリプト名を指定する必要があります。 スクリプトを選択した後、 読み込み ボタンを押して、スクリプトのすべての要件を読み込みます。スクリプトで 必須 または オプション として指定したパラメータは、定義するためのインターフェースに表示されます。 これらの値は、認証スクリプトの提供例で見られるように、認証中にスクリプト内で使用できます。 認証スクリプトで必要なパラメータを変更した後は、必ずスクリプトを再読み込みしてください。 そうしないと、インターフェースに表示されるパラメータが認証中に使用されるものと異なり、エラーが発生する可能性があります。 認証処理中に各ユーザーで使用される 認証情報 も認証スクリプトで指定でき、 ユーザー タブで設定します。 詳細...

認証検証

検証戦略

必要な 認証検証戦略 を選択してください。

ログイン済みメッセージを識別する正規表現パターン

この正規表現パターンが指定され、指定されたメッセージ内の任意の文字列と一致する場合、ZAPはこのメッセージをログイン済みとして扱います。 例えば、このパターンは http://example.com/logout リンクや「ようこそ、ユーザーXさん」メッセージの存在と一致させるために使用できます。

ログアウト済みメッセージを識別する正規表現パターン

この正規表現パターンが指定され、指定されたメッセージ内の任意の文字列と一致する場合、ZAPはこのメッセージをログアウト済みとして扱います。 例えば、このパターンは http://example.com/login リンクや「ようこそ、ゲストさん」メッセージの存在と一致させるために使用できます。

ポーリング頻度

これは、ZAPがユーザーがまだ認証されているかどうかを確認するためにポーリングする頻度を制御します。 この整数値は、選択したプルダウンに応じて、ZAPが再度ポーリングするまでに実行されるリクエスト数または秒数を表します。
このオプションは「指定したURLをポーリングする」検証戦略が選択されている場合のみ関連します。

検証用ポーリングURL

これは、ZAPがユーザーがまだ認証されているかどうかを確認するためにポーリングするURLです。
このオプションは「指定したURLをポーリングする」検証戦略が選択されている場合のみ関連します。

ポーリングリクエストのPOSTデータ

指定された場合、これはポーリングリクエストでPOSTとして送信されます。空の場合、ZAPはGETリクエストを実行します。
このオプションは「指定したURLをポーリングする」検証戦略が選択されている場合のみ関連します。

追加のポーリングリクエストヘッダー

ポーリングリクエストと一緒に送信されるオプションのヘッダーセットです。各行は、コロンで区切られた1つのヘッダーと値である必要があります。例：
Content-Type: application/json
X-Requested-With: XMLHttpRequest
このオプションは「指定したURLをポーリングする」検証戦略が選択されている場合のみ関連します。
