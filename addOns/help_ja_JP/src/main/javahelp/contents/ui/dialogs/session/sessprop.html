セッション プロパティダイアログ

このダイアログでは、セッションのプロパティを設定します。以下の画面から構成されています：

全般

セッションの名前と説明を設定します。

プロキシから除外

ローカルプロキシで無視するURLを管理します。

スキャナーから除外

スキャナーで無視するURLを管理します。

スパイダーから除外

スパイダー (標準およびAJAX) で無視するURLを管理します。

コンテキスト

コンテキストを管理するための一連の画面です。

URL 正規表現

〜から除外 ダイアログでは、正規表現を入力して除外対象のURLを定義できます。単一のメタ文字はバックスラッシュ (\) でエスケープできますが、 \Q...\E エスケープシーケンスも使用できます。 \Q と \E の間のすべての文字はリテラル文字として解釈されます。例えば、\Q*\d+*\Eはリテラルテキスト*\d+*と一致します。 このエスケープシーケンスは、一部のコンテキストメニューからURLを除外する際にZAPで使用されます。
注意: URLに「\E」が含まれている場合、 \Q...\E エスケープシーケンスを使用する際は以下の手順に従ってください：
  • エスケープシーケンスを開く
  • 「文字」\Eの前でエスケープシーケンスを閉じる
  • バックスラッシュをエスケープする
  • 「\E」の後で別のエスケープシーケンスを開く
  • 通常通りエスケープシーケンスを閉じる

例: subdomain.example.com/path?a=\E&moredata=2 は次のように表記する必要があります \Qsubdomain.example.com/path?a=\E\\E\Q&moredata=2\E
