セッションコンテキスト画面

これらの画面ではコンテキストを管理できます。 .

定義したコンテキストごとに画面セットが存在します。

トップ画面

これによりコンテキスト名と説明を設定できます。

コンテキストに含める

これによりコンテキストに含めるURLを管理できます。
いずれの正規表現にも一致しないURLは、コンテキストに含まれません。
注: 正規表現はURL全体と一致する必要があります。
ノート デフォルトポート (httpの場合は80、httpsの場合は443) で動作するターゲットをテストする場合、URLのコロン付きポート部分は含めないでください。 その部分を含めた場合 (例: http://example.com:80)、ターゲットのクロールやテストができなくなる可能性があります。 「デフォルト」ポートが指定されている場合、ブラウザとZAPはそのポート番号を省略して扱います。その結果、コンテキスト内で期待されるURLと一致せず、コンテキストの一部として操作できなくなります。

コンテキストから除外

これによりコンテキストから除外するURLを管理できます。
含めたくないが、1つ以上の「含める」正規表現に一致するURLについてのみ正規表現を指定する必要があります。
注: 正規表現はURL全体と一致する必要があります。

構造

セッションコンテキスト構造画面を参照してください.

テクノロジー

これにより、既知の場合、コンテキストで使用されるテクノロジーを指定できます。
デフォルトでは、すべてのテクノロジーが含まれます。
使用されていないことが分かっているテクノロジーを除外すると、除外されたテクノロジーに固有のルールをスキップできるため、 アクティブスキャン が高速化される可能性があります。

セッションコンテキスト認証画面を参照してください.

スクリプトベースセッション管理

このセッション管理方式を使用するには、 まず スクリプト タブを使用して スクリプト を作成 (および保存) する必要があります (スクリプトタブにはこのタイプのテンプレートや例が用意されています)。 次に、ドロップダウンリストでスクリプト名を指定する必要があります。 スクリプトを選択した後、 読み込み ボタンを押して、スクリプトのすべての要件を読み込みます。スクリプトで 必須 または オプション として指定したパラメーターは、定義するためのインターフェースに表示されます。 これらの値は、セッション管理スクリプトの提供例で見られるように、セッション管理処理中にスクリプト内で使用できます。 セッション管理スクリプトで必要なパラメーターを変更した後は、必ずスクリプトを再読み込みしてください。 そうしないと、インターフェースに表示されるパラメーターがセッション管理処理中に使用されるものと異なり、エラーが発生する可能性があります。 詳細...

ユーザー

これによりアプリケーション全体のさまざまなアクションで使用される可能性のあるユーザーセットを設定できます。

ユーザーの認証情報セクションは、コンテキスト用に設定された認証方式によって異なります。

カスタム ページ

この機能により、404 - Not Found、500 - Internal Server Error などの条件において、ターゲットがRFC準拠または期待される誤りレスポンスを返さない状況を処理できます。 多くの場合、開発者はカスタム404または500処理を実装しますが、実際には適切なステータスコードを返しません。 カスタムページ定義を活用することで、レスポンスに関連するステータスコードに関係なく、エラーレスポンスを識別するためのURLやレスポンスコンテンツを定義できます。 詳細...

URL正規表現

* に含める, * から除外 パネル、および認証パネルの ログイン/アウト指標 などで、除外URLを定義するための正規表現を入力できます。 単一のメタ文字をバックスラッシュでエスケープできますが、 \Q...\E エスケープシーケンスも使用できます。 \Q と \E の間のすべての文字はリテラル文字として解釈されます。例えば、\Q*\d+*\Eはリテラルテキスト*\d+*と一致します。 このエスケープシーケンスは、一部のコンテキストメニューからURLを除外する際にZAPで使用されます。
注: URLに「\E」が含まれている場合、 \Q...\E エスケープシーケンスを使用する際は以下の手順に従ってください：
  • エスケープシーケンスを開く
  • 「文字」\Eの前でエスケープシーケンスを閉じる
  • バックスラッシュをエスケープする
  • 「\E」の後で別のエスケープシーケンスを開く
  • 通常通りエスケープシーケンスを閉じる

例: subdomain.example.com/path?a=\E&moredata=2 は次のように表記する必要があります \Qsubdomain.example.com/path?a=\E\\E\Q&moredata=2\E
