アラート

アラートは、潜在的な脆弱性と、特定のリクエストに関連付けられています。
リクエストは1 つ以上のアラートを持つことができます。

アラートはリスクを示すフラグを UI 上に表示します:
      High
      Medium
      Low
      Informational
      False Positive（誤った警告）

アラートは、アクティブスキャン、パッシブスキャン、スクリプト、拡張アドオン、または「アラートの追加」ダイアログを使用して手動で作成するなど、さまざまなZAPコンポーネントによって生成されます。 「アラートの追加」ダイアログを使用して、アラートの詳細/情報を更新または変更することもできます。

アラートは履歴タブの中に最もリスクとなるアラートを示します。
全てのアラートはアラートタブに列挙され、そのリスクの総数はフッターにカウントされます。.

アラートの上書き

ZAPによって生成されたアラートには、発生したアラートに関する一般的な情報と特定の情報の両方が含まれます。 特定の情報は、URLや影響を受けるパラメータなど、潜在的に見つかった問題に直接関連しています。 一般的な情報には、説明や関連するオンラインリソースへのリンクなどが含まれます。

「alert override 構成ファイル」を使用して、一般的な情報を置換または追加することができます。 これにより、義務付けられたポリシー、内部リンク、または使用する特定のテクノロジのアドバイスなど、会社固有の情報を含めることができます。

「alert override」設定ファイルは、変更したい情報だけを含むUTF-8形式のファイルです。 '#'で始まる行はコメントとして扱われ、無視されます。

フォーマットは以下要領です。 

<alertid>.<property> = [ + | - ] <your information>
以下プロパティーがあります。
  • name
  • description
  • solution
  • otherInfo
  • reference
The format for alert tags is: 
<alertid>.tag.<key> = [ + | - ] [ <value> ]
Note that:
  • Spaces in the tag key must be escaped with a backslash.
  • The tag value is optional.
For example 
# 40012 = Reflected XSS
40012.solution = Follow our company specific guidelines at http://internet.example.com/xss.html
40012.tag.Bug\ Tracker\ URL = https://example.com/issues/1234
If the value starts with a '+' then it is appended to the existing information.
If it starts with a '-' then it is prepended to the existing information.
If it does not start with a '+' or '-' then it replaces the existing information.

アラートオーバーライド設定ファイルは、APIで指定できます。 　指定方法は次の通り、, アラート画面のオプション または コマンドライン オプション 

-config alert.overridesFilename=<filename>
