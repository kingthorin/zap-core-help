ブレークポイント

ブレークポイントは、ブラウザからWebアプリケーションへのリクエスト、およびアプリケーションからブラウザへのレスポンスを途中で捕捉し、変更できる機能です。
捕捉されたリクエストやレスポンスはブレークタブに表示され、そこで内容を編集できます。これにより、無効化されているフィールド (disabledフィールド) や非表示のフィールド (hiddenフィールド) を変更したり、JavaScriptなどで実装されるクライアント側の検証をバイパスしたりすることができます。
ブレークポイントは、侵入テストにおいて不可欠な技術です。

トップレベルツールバーのボタンを使用して、リクエストまたはレスポンスに対する「グローバル」ブレークポイントを設定できます。
グローバルブレークポイントを有効にすると、すべてのリクエスト (またはレスポンス) がZAPによって捕捉され、送信する前に任意の変更を加えることができます。

サイトタブおよび履歴タブの「ブレーク…」右クリックメニューと、トップレベルツールバーの「カスタムHTTPブレークポイントを追加」ボタンを使用して、特定の条件に基づくブレークポイントを設定することもできます。
これらの条件に一致するリクエストとレスポンスのみがZAPによって捕捉されます。
カスタムブレークポイントはブレークポイントタブに表示されます。

ブレークポイントのオプションは、オプション ブレークポイント画面で設定します.
