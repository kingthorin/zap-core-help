検証戦略

認証検証戦略は、ZAPがユーザーが現在認証されているか (ログイン中か)、それとも未認証か (ログアウト中か) を判断する方法を定義します。ZAPは複数の検証戦略をサポートしており、アプリケーションの特性に応じて最適なものを選択できます。

すべてのレスポンスを確認する

この戦略を使用すると、ZAPはすべてのレスポンスに対して指定された正規表現パターンを適用し、認証状態を判断します。 これは通常、完全なHTMLページを返す従来型のWebアプリに役立ちます。

すべてのリクエストを確認する

この戦略を使用すると、ZAPはすべてのリクエストに対して指定された正規表現パターンを適用し、認証状態を判断します。 これは通常、JWTなどのセッション状態をクライアント側で維持し、リクエストごとに送信するモダンなWebアプリ (クライアントサイドレンダリング、SPA) に役立ちます。

すべてのリクエストまたはレスポンスを確認する

この戦略を使用すると、ZAPはすべてのメッセージのリクエストとレスポンスの両方に対して指定された正規表現パターンを使用します。

指定したURLをポーリングする

この戦略を使用すると、ZAPは指定されたURLを設定された頻度で定期的にポーリング (アクセス) します。 すべてのメッセージの認証状態は、最後のポーリングリクエストの結果に基づいて判断されます。 指定された正規表現パターンは、ポーリングリクエストのレスポンスに対してのみ使用されます。 ユーザーがログインしているかログアウトしているかを確実に判断できるURLが存在するアプリケーションに適しています。

正規表現パターン

すべての戦略は、指定されたメッセージがログインしているかどうかを判断するために2つの正規表現パターンを使用します。 いずれか一方または両方の正規表現を指定できます - どちらも指定しない場合、認証戦略は使用されません。

ログイン/ログアウト指標のいずれかを設定するには、セッションコンテキスト認証画面ダイアログ -> 認証パネル -> ログイン/ログアウト指標フィールドに正規表現を直接入力するか、サイトツリーまたは履歴タブで認証済みメッセージを見つけて選択し、レスポンスビューを開き、マウスを使用して指標として定義するテキストを選択し、「コンテキストとしてフラグ付け... ログイン/ログアウト済み指標」右クリックメニューオプションを選択します。
