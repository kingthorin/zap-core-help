ユーザー

ZAPにおける「ユーザー」とは、テスト対象のWebアプリケーションにログインする実際のユーザーアカウントを表す概念です。 ユーザーを定義することで、認証が必要なページや機能に対してセキュリティテストを実行できます。 各コンテキストには、複数のユーザーを定義でき、異なるユーザーアカウント (例: 一般ユーザー、管理者、モデレーターなど) の視点からテストを実行できます。 最も一般的には、さまざまなスキャン中に、リクエストメッセージをユーザーの視点から送信できます。

ユーザーの概念は、セッション管理と認証の概念と密接に結びついています。 ユーザーがZAP内で最初に使用されると、(コンテキストに定義された認証方式に従って) 認証が実行され、 (コンテキストに定義されたセッション管理に従って) このユーザー用のセッションが作成・設定されます。 その後、ユーザーの視点から送信されるリクエストは、Webサーバーがそれらを認証されたWebアプリ/Webサイトのユーザーによって送信されたものとして識別できるように、(必要に応じて) 変更および送信されます。 ユーザーが未認証に見える場合 (認証検証戦略によって判断される)、新しい認証が実行され、それに応じてセッションが更新されます。

Webサイト/Webアプリでユーザーの認証を実行するために、認証方式は認証の実行方法 (プロセス) を定義しますが、必要な認証情報 (正確な識別子) はユーザーに依存するため、ZAPではユーザーで設定されます。
