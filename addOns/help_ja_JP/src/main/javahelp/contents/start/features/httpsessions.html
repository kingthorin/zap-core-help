HTTPセッション

このツールは、特定のサイトに存在するHTTPセッションを追跡し、ZAPユーザーがすべてのリクエストで特定のセッションを強制的に使用できるようにします。 要するに、ユーザーは既存のセッションを「破棄」することなく、サイトのユーザーセッションを簡単に切り替えたり、新しいセッションを作成したりできます。

このツールは手動テストのみを対象としており、自動化で使用するべきではありません。自動化には、はるかに優れた代替手段が存在します。 ZAPの自動化を初めて使用する場合は、ZAPの 認証ディシジョンツリー (外部リンク) から始めるのが最適です。

このツールは、セッショントークンの概念に基づいています。セッショントークンとは、HTTPサーバーがリクエストメッセージを過去のリクエストや保存されたデータと関連付けることを可能にするHTTPメッセージパラメーター (現時点はCookieのみ) です。 ZAPの概念では、セッショントークンは「デフォルトセッショントークン」と「サイトセッショントークン」の2つのカテゴリに分類されています。 デフォルトセッショントークンは、ユーザーがオプション画面で設定できるトークンであり、デフォルトですべてのサイトのセッショントークンとして自動的にみなされるものです (例: phpsessid, jsessionid など)。 サイトセッショントークンは、特定のサイト用の一連のトークンであり、通常はパラメータータブで利用可能なポップアップメニューを使用して設定されます.

このツールは、定義済みのセッショントークン、または自動検出されたデフォルトセッショントークンを使用して、通信に存在するHTTPセッションを自動的に検出します。 検出されたセッションはHTTPセッションタブに表示されます.

ユーザーは、HTTPセッションタブにあるボタンを使用して、既存のセッションを破棄することなく新しいセッションを作成したり、いずれかのセッションを「アクティブ」として強制したりできます。 あるセッションが「アクティブ」になると、対応するサイトに送信されるすべてのリクエストが変更され、セッショントークンはアクティブなセッションに一致するように設定されます。 これにより、ユーザーは特定のメッセージをあるセッションの「一部」として強制的に送信し、その後別のセッションに切り替えてメッセージを送信することが容易になります。

HTTPセッションツールは、HTTPセッションオプション画面で設定します.
