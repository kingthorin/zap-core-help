セッション管理

ZAPは、Webサイト/Webアプリに使用できる複数のタイプのセッション管理 (セッション管理方式と呼ばれます) を処理します。 各コンテキストには、セッションの維持方法を規定するセッション管理方式が定義されています。

Cookieベースセッション管理

この方式では、セッションはCookieを通じて追跡されます。セッショントークン (セッションを識別するための値) は、HTTPセッション拡張機能から自動的にインポートされます。

HTTP認証セッション管理

この方式では、セッションはHTTPリクエストヘッダーのAuthorizationフィールドを使用して管理されます。Basic認証、Digest認証、NTLM認証などで使用されます。.

スクリプトベースセッション管理

この方式は、セッション管理がより複雑で、セッション管理プロセスを処理するカスタムスクリプトが有益なWebサイト/Webアプリに役立ちます。 この方式を使用するには、まず、Webアプリケーションで必要に応じてメッセージを送信したり、他のアクションを実行したりするセッション管理スクリプトを定義する必要があります。 このスクリプトは、特定のコンテキストで使用するために選択され、セッション管理アクションが実行されるたびに呼び出されます。 設定はセッションコンテキスト ダイアログを使用して行うことができます。この機能を使用するには、マーケットプレイスからスクリプトコンソールアドオンをインストールする必要があります。

設定方法
