コンテキスト

コンテキストは、関連するURLをグループ化してまとめて管理する仕組みです。
任意のコンテキストを定義できますが、コンテキストは通常、Webアプリケーションに対応することが想定されています。
テストしているシステムを構成する各Webアプリケーションに対して新しいコンテキストを定義し、それぞれをテストする際にスコープに設定することを推奨します。

コンテキストは、サイトツリー内のすべてのURLに適用される一連の正規表現 (RegEx) として定義されます。
注: 正規表現はURL全体に一致する必要があります。

コンテキストは以下の方法で設定できます：

  • サイトタブまたは履歴タブの右クリックメニュー項目
  • セッションコンテキスト ダイアログ.
