Scripts

O ZAP suporta scripts que podem ser incorporados ao próprio ZAP e acessar as estruturas de dados internas e classes do ZAP. Esses scripts permitem que você aprimore dinamicamente o ZAP de dentro do próprio ZAP.

ZAP supports any scripting language that supports JSR 223, including:

  • ECMAScript / JavaScript (through the GraalVM JavaScript add-on)
  • Zest
  • Groovy https://groovy-lang.org/
  • Kotlin https://kotlinlang.org/
  • Python https://www.jython.org
  • Ruby - https://jruby.org/
  • e muitos outros...
AVISO - Os scripts são executados com as mesmas permissões que o ZAP, portanto, não execute scripts que você não confia!

Tipos de scripts

Diferentes tipos de scripts são suportados:
  • Active Rules - esses scripts são executados como parte do Active Scanner e podem ser habilitados individualmente.
  • Autenticação - scripts que são invocados quando a autenticação é realizada para um Contexto. Para serem utilizados, eles precisam ser selecionados ao configurar o Método de Autenticação Baseado em Script para um Contexto.
  • Extender - scripts que podem adicionar novas funcionalidades, incluindo elementos gráficos e novos pontos de extremidade da API.
  • HTTP Sender - scripts que são executados em todas as requisições/respostas enviadas/recebidas pelo ZAP. This includes the proxied messages, messages sent during active scan, fuzzer, ...
  • Input Vector - scripts para definir exatamente o que o ZAP deve atacar. Eles também podem alterar como qualquer URL é representada na estrutura de sites (Sites Tree) do ZAP.
  • Passive Rules - esses scripts são executados como parte do Passive Scanner e podem ser habilitados individualmente.
  • Proxy - esses scripts são executados "inline", ou seja, podem alterar cada requisição e resposta e podem ser habilitados individualmente. Eles também podem acionar pontos de interrupção (breakpoints).
  • Gerenciamento de Sessão - scripts que definem como uma aplicação gerencia as sessões.
  • Stand Alone - scripts que são independentes e são executados apenas quando você os inicia manualmente.
  • Targeted - scripts que são invocados com uma URL de destino e são executados apenas quando você os inicia manualmente.
Nota: Os complementos podem adicionar tipos adicionais de scripts, os quais devem ser descritos na documentação do respectivo complemento.

Para obter mais detalhes sobre como executar scripts no ZAP, consulte as páginas de ajuda do complemento Script Console.
