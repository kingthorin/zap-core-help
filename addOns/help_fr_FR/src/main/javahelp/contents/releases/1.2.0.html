Release 1.2.0

Les modifications suivantes ont été apportées dans cette version :

Significant changes:

Memory Leaks

Les fuites de mémoire ont été corrigés dans la scanner actif and spider.

Invoke applications

External applications can now be invoked from the Sites and History tabs.

Passive Scanner

The passive scanner now looks for vulnerabilities, such as:
    La saisie semi-automatique des formulaires avec des champs de mot de passe
    Cookies sans le "HttpOnly" drapeau
    SSL Cookies sans le "sécuriser" drapeau
    La faiblesse de l'authentification

Changements mineurs :

XML Reports

A new 'Generate XML Report...' menu item is now included in the top level Reports menu.

Manuel de l'Éditeur de Requête et Renvoyer les boîtes de dialogue

Requests submitted by the Manual Request Editor and Resend dialogs are now shown in the Sites and History tabs.
A new 'Method' pull down allows you to switch between the HTTP methods, this automatically moves parameters between the URL and the body when a POST method is selected or deselected.

Alerts UI

The Sites tab now shows any alerts as flags to the right of any node names.
Le nombre d'alertes dans le pied de page affiche maintenant le nombre des différents types d'alertes plutôt que le nombre total de cas.

Active scanner l'option retard

Le délai en millisecondes entre chaque active scanner request can now be set via the Options Active Scan screen. This will increase the time an active scan takes but will reduce the load on the target.

Modifications UI

The Sites tab now takes up all of the left hand side - this can be changed back via the Options Display screen if required.

La barre d'outils sur les onglets Request, Response et Break tabs and the Manual Request Editor and Resend dialogs is now at the top rather than the bottom.

The scanner counts in the footer are now displayed on the right hand side.
